# Theory

Theory documents what must be true about duraclaw — zero implementation references (no internal file paths, no class names, no import paths), with a carve-out for named external dependencies in `boundaries.md` and where their existence shapes an invariant elsewhere. The 6 categorical files below are fixed.

New theory content must fit one of these categories — if it doesn't, the categories need revision, not a new file.

## The 6 categorical files

- `domains.md` — entity types and lifecycles
- `data.md` — state-authority model
- `dynamics.md` — how work moves
- `topology.md` — system layout
- `trust.md` — token boundaries
- `boundaries.md` — external dependencies

These files do not exist yet — they are authored in P1 of GH#135. The list above is the closed set; adding a 7th category requires revising this discipline, not appending a file.
