# @duraclaw/mobile — Capacitor Android shell

Capacitor 8 wrapper around the orchestrator React bundle. The APK ships
the same UI as the web app, but as a sandboxed `capacitor://localhost`
WebView that talks to the cloud Worker over HTTPS / WSS. There is **no
local server in the APK** — this is a thin client that renders the
deployed orchestrator.

## Architecture (one paragraph)

The orchestrator's Vite bundle (`apps/orchestrator/dist/client`) is
copied into `apps/mobile/android/app/src/main/assets/public/` by
`cap sync`. The Capacitor runtime hosts the WebView, exposes native
plugin bridges (sqlite, push, app lifecycle, network, preferences), and
intercepts requests so relative `/api/*` paths get rewritten to the
Worker URL baked into the bundle at build time. Platform branches in the
React code are gated by `import.meta.env.VITE_PLATFORM === 'capacitor'`
so Vite tree-shakes Capacitor imports out of the web bundle entirely
(verified — no `@capacitor/*` strings in the web `dist/`).

Three things differ from the web build:

| Concern | Web build | Capacitor build |
|---|---|---|
| Persistence | OPFS via wa-sqlite | Native sqlite via `@capacitor-community/sqlite` |
| Auth storage | Cookies (same-origin) | Bearer token via `better-auth-capacitor` (`withCapacitor()`) |
| Push | Web Push (VAPID + service worker) | FCM HTTP v1 → `@capacitor/push-notifications` |
| API origin | Relative (`window.location.origin`) | Absolute (`VITE_API_BASE_URL`) |
| WS host | Relative (`window.location.host`) | Absolute (host of `VITE_WORKER_PUBLIC_URL`) |

All gating lives in `apps/orchestrator/src/lib/platform.ts`
(`isNative()`, `apiBaseUrl()`, `apiUrl()`, `wsBaseUrl()`).

## Prerequisites

- **JDK 21** (Capacitor 8 default toolchain). Set `JAVA_HOME` to the JDK 21 install:
  ```bash
  export JAVA_HOME=/usr/lib/jvm/java-21-openjdk-amd64
  ```
- **Android SDK** with these packages (sized ~750 MB total):
  - `cmdline-tools;latest`
  - `platform-tools` (37+)
  - `platforms;android-36`
  - `build-tools;36.0.0`
- Env vars on PATH:
  ```bash
  export ANDROID_HOME="$HOME/Android/sdk"
  export ANDROID_SDK_ROOT="$ANDROID_HOME"
  export PATH="$ANDROID_HOME/cmdline-tools/latest/bin:$ANDROID_HOME/platform-tools:$ANDROID_HOME/build-tools/36.0.0:$PATH"
  ```

The version pins are not arbitrary:
- `apps/mobile/android/variables.gradle` declares `compileSdk = 36` /
  `targetSdk = 36`. Building with anything older than `platforms;android-36`
  or `build-tools;36.0.0` fails at AAPT2.
- `apps/mobile/android/app/capacitor.build.gradle` (auto-generated by
  `cap sync`) sets `JavaVersion.VERSION_21`. JDK 17 fails compilation.

### One-time fresh-machine setup

```bash
# 1. Install JDK 21 (Ubuntu)
sudo apt install -y openjdk-21-jdk

# 2. Install Android cmdline-tools (download from
#    https://developer.android.com/studio#command-line-tools-only)
mkdir -p ~/Android/sdk
unzip commandlinetools-linux-*.zip -d ~/Android/sdk/cmdline-tools-tmp
mv ~/Android/sdk/cmdline-tools-tmp/cmdline-tools ~/Android/sdk/cmdline-tools/latest
rmdir ~/Android/sdk/cmdline-tools-tmp

# 3. Set env (also append to ~/.bashrc — see "Persisting env" below)
export ANDROID_HOME=$HOME/Android/sdk
export ANDROID_SDK_ROOT=$ANDROID_HOME
export PATH=$ANDROID_HOME/cmdline-tools/latest/bin:$PATH

# 4. Accept licenses + install SDK packages
yes | sdkmanager --sdk_root=$ANDROID_HOME --licenses
sdkmanager --sdk_root=$ANDROID_HOME --install \
  "platforms;android-36" "build-tools;36.0.0" "platform-tools"

# 5. Verify
sdkmanager --sdk_root=$ANDROID_HOME --list_installed
```

## Build

From repo root:

```bash
pnpm --filter @duraclaw/mobile build:android
```

This runs `apps/mobile/scripts/build-android.sh`, four phases:

1. **Source `apps/mobile/.env.production`** — exports `VITE_PLATFORM=capacitor`
   and the Worker URLs into the build env so Vite substitutes them at
   compile time.
2. **`pnpm --filter @duraclaw/orchestrator build`** — emits
   `apps/orchestrator/dist/client/`.
3. **`pnpm exec cap sync android`** — copies `dist/client/` into
   `android/app/src/main/assets/public/` and regenerates
   `capacitor.build.gradle` + `capacitor.settings.gradle` to register
   installed plugins.
4. **`./gradlew assembleRelease`** in `android/` — produces
   `android/app/build/outputs/apk/release/app-release-unsigned.apk` (~30 MB).

Build time on a warm cache: ~90 seconds. First run downloads Gradle 8.x
and AGP dependencies (~5 minutes, one time).

### `.env.production`

Located at `apps/mobile/.env.production`. **Only `VITE_*`-prefixed
values** — Vite inlines them into the public JS bundle, which means
they're visible in the APK and git history. Server secrets belong in
wrangler secrets, never here.

```
VITE_PLATFORM=capacitor
VITE_API_BASE_URL=https://duraclaw.workers.dev
VITE_WORKER_PUBLIC_URL=https://duraclaw.workers.dev
```

`VITE_API_BASE_URL` is the HTTP origin for `apiUrl()` rewrites;
`VITE_WORKER_PUBLIC_URL` supplies the WSS host for partysocket. Currently
the same URL but kept as separate exports because they may diverge
(e.g. CDN-fronted HTTP, direct-Worker WS).

## Sign

```bash
export KEYSTORE_PATH=~/.duraclaw-keystore.jks
export KEYSTORE_PASS=<pass>
export KEY_ALIAS=duraclaw
export KEY_PASS=<pass>
apps/mobile/scripts/sign-android.sh
```

Outputs `app-release-signed.apk` next to the unsigned APK and runs
`apksigner verify --print-certs` as a sanity check. Signs with v2 + v3
schemes (Android 7.0+).

If env vars are unset, the script prints a complete keystore-generation
help block and exits non-zero.

### Dev keystore (local sideload only)

A throwaway keystore for sideloading to your own dev device. **Never
ship APKs signed with this — installing one device-side blocks future
upgrades signed with any other key.**

```bash
keytool -genkey -v -keystore ~/.duraclaw-dev-keystore.jks \
  -keyalg RSA -keysize 2048 -validity 10000 -alias duraclaw \
  -storepass devpassword -keypass devpassword \
  -dname "CN=Duraclaw Dev, OU=Eng, O=Baseplane, L=Local, S=Local, C=US"

export KEYSTORE_PATH=~/.duraclaw-dev-keystore.jks
export KEYSTORE_PASS=devpassword
export KEY_ALIAS=duraclaw
export KEY_PASS=devpassword
apps/mobile/scripts/sign-android.sh
```

### Production keystore

Stored in the **1Password Engineering vault** as item
`duraclaw-android-keystore`. Workflow:

```bash
op item get duraclaw-android-keystore --field keystore --reveal > /secure/tmp/duraclaw-prod.jks
export KEYSTORE_PATH=/secure/tmp/duraclaw-prod.jks
export KEYSTORE_PASS="$(op read 'op://Engineering/duraclaw-android-keystore/keystore_password')"
export KEY_ALIAS=duraclaw
export KEY_PASS="$(op read 'op://Engineering/duraclaw-android-keystore/key_password')"
apps/mobile/scripts/sign-android.sh
shred -u /secure/tmp/duraclaw-prod.jks
```

Rotation invalidates upgrades for every installed APK — rotate **only**
with explicit infra approval.

For CI: same env vars, supplied via the runner's secret bindings. Never
commit the `.jks` file or hardcode passwords.

## Install on device

1. On the Android device, enable **Developer Options** (tap Build Number
   7×) and **USB debugging** (or **Install from unknown sources** for
   non-ADB sideload).
2. Connect via USB and confirm the device is visible:
   ```bash
   adb devices
   ```
3. Install:
   ```bash
   adb install -r apps/mobile/android/app/build/outputs/apk/release/app-release-signed.apk
   ```
   For non-ADB distribution: upload to a 1Password shared item or signed
   S3 URL, send the link, tap-to-install.
4. First launch prompts for notification permission (FCM push).
5. Tail logs while the app runs:
   ```bash
   adb logcat | grep -iE 'duraclaw|capacitor'
   ```

## FCM (Firebase Cloud Messaging) setup

The push pipeline ships in code (P1.5) but needs Firebase project
provisioning before notifications actually deliver:

1. Create a Firebase project at console.firebase.google.com.
2. Add an Android app with package ID `com.baseplane.duraclaw`.
3. Download `google-services.json`, place at
   `apps/mobile/android/app/google-services.json`. **Gitignored** —
   never commit (it's not a secret per se but pollutes the repo).
4. Generate a service account key (Project Settings → Service Accounts
   → Generate new private key). Save the JSON.
5. Set the Worker secret:
   ```bash
   wrangler secret put FCM_SERVICE_ACCOUNT_JSON
   # paste the entire service account JSON when prompted
   ```
6. The orchestrator's `SessionDO` reads `env.FCM_SERVICE_ACCOUNT_JSON`,
   mints a short-lived OAuth2 access token via `jose` (RS256 JWT), and
   POSTs to `https://fcm.googleapis.com/v1/projects/<project-id>/messages:send`.
   `sendFcmNotification()` (in `apps/orchestrator/src/lib/push-fcm.ts`)
   handles 404/410 token-gone responses by deleting the row from the
   `fcm_subscriptions` D1 table.

If `FCM_SERVICE_ACCOUNT_JSON` is unset, the FCM fan-out branch in the DO
is skipped silently — push is degraded but the rest of the app works.

## Cap sync — what's auto-generated, what's owned

After `pnpm exec cap sync android`, these files are **regenerated every
time** — never hand-edit:

- `apps/mobile/android/app/src/main/assets/public/**` (the web bundle)
- `apps/mobile/android/app/capacitor.build.gradle` (plugin registration,
  Java version pin)
- `apps/mobile/android/app/capacitor.settings.gradle` (plugin module
  paths into pnpm store)
- `apps/mobile/android/capacitor.settings.gradle`

These are owned by us and committed:

- `apps/mobile/capacitor.config.ts` — root Capacitor config (appId,
  webDir, plugin opts)
- `apps/mobile/android/variables.gradle` — SDK version pins
- `apps/mobile/android/app/build.gradle` — app-level gradle
- `apps/mobile/android/app/src/main/AndroidManifest.xml` — permissions,
  intents, activity flags
- `apps/mobile/android/app/src/main/res/values/colors.xml` — theme colors
  (referenced from `styles.xml`)
- `apps/mobile/android/app/src/main/res/values/strings.xml` — app name,
  display strings
- `apps/mobile/android/app/src/main/res/xml/file_paths.xml` — scoped
  FileProvider paths

Adding a new Capacitor plugin:

```bash
pnpm --filter @duraclaw/mobile add @capacitor/<plugin>
cd apps/mobile && pnpm exec cap sync android
```

`cap sync` will rewrite the generated gradle files to register the new
plugin. Commit the regenerated files alongside the package.json change.

## Updating the SDK

```bash
sdkmanager --sdk_root=$ANDROID_HOME --update
```

Bumping `compileSdkVersion` / `targetSdkVersion`: edit
`apps/mobile/android/variables.gradle`, install the matching
`platforms;android-NN` and `build-tools;NN.0.0`, update this README's
version table and the `.bashrc` PATH entry.

## Common failures

| Symptom | Cause | Fix |
|---|---|---|
| `error: resource color/colorPrimary not found` at AAPT2 | Missing `colors.xml` (was lost from cap-add scaffold) | Restore `apps/mobile/android/app/src/main/res/values/colors.xml` |
| `Unable to find Java major version` | `JAVA_HOME` unset or pointed at JDK 17 | `export JAVA_HOME=/usr/lib/jvm/java-21-openjdk-amd64` |
| `Failed to find target with hash string 'android-36'` | Missing platform | `sdkmanager --install "platforms;android-36"` |
| `apksigner: command not found` | `build-tools/36.0.0` not on PATH | `export PATH=$ANDROID_HOME/build-tools/36.0.0:$PATH` |
| WebView loads but `/api/*` returns 404 | `VITE_API_BASE_URL` unset, falls back to `capacitor://localhost` | Check `.env.production` is sourced; rebuild |
| WS never connects, console shows `wss://localhost/...` | `VITE_WORKER_PUBLIC_URL` unset | Same fix; partysocket falls back to `window.location.host` |
| App backgrounded for 30s and reopens with stale messages | Working as designed — see `use-app-lifecycle.ts` (5 s background-close timer + foreground hydrate via `getMessages` RPC) |
